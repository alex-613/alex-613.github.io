Augmented reality for malleable robot machine human interaction

Typically, the majority of tasks performed by serial robot arms only require 2 or 3 degrees of freedom. However, as with any robot, there will be singularities that exist with manipulating those robots. In addition, with increased joints, the weight, cost, size, power, points of failure and complexity of design greatly increases as well, so a simpler robot is typically preferred over a complex one.

To mitigate some of these issue, soft robotics does provide a plausible solution to this problem, however it is limited by the complixity of its control system and motion planning. With this rationale in mind, the REDS lab has developed a compromise between the two, which is malleable robots.

Malleable robots allows the user to configure the robot to a particular shape depending on the workspace required. Now, configurrations such as articulated, SCARA and spherical can be easily achieved. However with that said, this system currently cannot be configured accurately, without the need for a visual feedback system that enables machine human interaction. This is where augmented reality comes in, where the physical environment can be overlaid with virtual holograms to allow the user to see the end pose of the robot, and reconfigure it accordingly.

Currently, I am designing an app for the Microsoft hololens using MRTK and Unity game engine, which does exactly that. Given the end pose and position of the malleable part of the robot, the Hololens will generate a configured hologram that gets the user to that point. It is then also able to provide feedback to the user whether they have correctly configured the robot or not.
